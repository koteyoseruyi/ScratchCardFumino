# ScratchCardFumino

一个功能丰富的 Minecraft 刮刮卡插件，支持多种刮刮卡、自定义界面与音效、全服公告等功能。

## 功能简介

- **多种刮刮卡**：默认包含铜卡、金卡、钻石卡三种，每种卡独立配置文件
- **自定义界面**：每种卡可独立配置界面大小、标题、背景材质、按钮位置、高亮材质
- **自定义音效**：每种卡可独立配置打开、刮奖、中奖、空奖、完成等音效
- **同类型加成**（金卡、钻石卡）：相同奖励类型每多出现一个，该类型奖金额外增加一份基础奖金
- **倍率系统**（钻石卡）：3 个倍率槽位，乘积放大最终奖金
- **全服公告**：金卡和钻石卡获得超高收益时，全服彩虹渐变公告
- **商店菜单**：`/scratchshop` 打开商店，翻页展示所有刮刮卡，左键点击购买
- **购买指令**：`/buycard` 指令购买，支持 Tab 自动补全
- **管理指令**：`/getcard` 管理员给予，支持 Tab 自动补全
- **自动发现**：插件自动扫描 `cards/` 目录下的 `.yml` 文件，新增卡片无需改代码

## 依赖

| 依赖 | 说明 |
|------|------|
| Paper 1.21+ | 服务端核心 |
| Vault | 经济系统接口（需搭配经济插件如 EssentialsX、CMI 经济等使用） |

## 指令

| 指令 | 权限 | 描述 |
|------|------|------|
| `/getcard <玩家> <种类> <数量>` | `scratchcard.admin`（默认 OP） | 管理员给予玩家指定刮刮卡 |
| `/buycard <种类> <数量>` | 无（所有玩家可用） | 玩家花费金币购买刮刮卡 |
| `/scratchshop` | 无（所有玩家可用） | 打开刮刮卡商店菜单 |

指令均支持 Tab 自动补全，输入时按 Tab 键可补全玩家名和卡片种类。

## 配置文件说明

### 全局配置（config.yml）

```yaml
multiplier: 500
```

- `multiplier`：全局定价倍率，所有卡的价格和奖金均乘以该值，不能小于 1，默认为 500

### 卡片配置（cards/ 目录下的 .yml 文件）

每个刮刮卡独立一个 `.yml` 文件，位于 `plugins/ScratchCardFumino/cards/` 目录下。插件启动时自动加载该目录下所有 `.yml` 文件。

#### 基础字段

| 字段 | 说明 |
|------|------|
| `display` | 卡片显示名称，支持 MiniMessage 格式 |
| `lore` | 卡片描述文本列表 |
| `slot-count` | 总槽位数 |
| `price` | 卡片基础价格（实际价格 = price × multiplier） |
| `bonus-enabled` | 是否启用同类型加成（true/false） |
| `reward-slots` | 奖励槽数量 |
| `multiplier-slots` | 倍率槽数量 |

#### 界面配置（ui 段）

| 字段 | 说明 |
|------|------|
| `size` | 界面格子数量（如 27=9×3，54=9×6） |
| `title` | 界面标题，支持 MiniMessage 格式 |
| `background-material` | 背景材质（Bukkit Material 枚举名） |
| `slot-positions` | 按钮槽位列表（整数数组，前 reward-slots 个为奖励槽，剩余为倍率槽） |
| `highlight-material` | 倍率槽四周高亮材质（可选，不配置则无高亮） |

#### 音效配置（sounds 段）

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `open` | 打开刮刮卡音效 | `BLOCK_ANVIL_LAND` |
| `reward` | 刮到奖励音效 | `ENTITY_VILLAGER_CELEBRATE` |
| `empty` | 刮到空奖音效 | `ENTITY_VILLAGER_HURT` |
| `scratch` | 每次刮卡音效 | `BLOCK_GRINDSTONE_USE` |
| `complete` | 全部刮完音效 | `ENTITY_FIREWORK_ROCKET_LAUNCH` |
| `multiplier-sound` | 倍率槽音效 | `BLOCK_NOTE_BLOCK_HARP` |
| `multiplier-pitch-min` | 倍率音效最低音高（1×时） | 0.6 |
| `multiplier-pitch-max` | 倍率音效最高音高（5×时） | 2.0 |

#### 奖励配置（rewards 段）

每个奖励类型包含以下字段：

| 字段 | 说明 |
|------|------|
| `weight` | 权重值（用于概率计算） |
| `reward` | 基础奖金（实际发放 = reward × multiplier） |

#### 倍率配置（multipliers 段，仅钻石卡需要）

每个倍率值包含以下字段：

| 字段 | 说明 |
|------|------|
| `weight` | 权重值（用于概率计算） |

## 默认配置概率公示(期望均为88%(±0.5%))

## 铜刮刮卡（3 个奖励槽，无加成）

| 奖励类型 | 权重 | 概率 | 奖金（元） |
|----------|------|------|------------|
| 空奖     | 47   | 47%  | 0          |
| 小奖     | 35   | 35%  | 0.30       |
| 中奖     | 12   | 12%  | 0.60       |
| 大奖     | 6    | 6%   | 2.00       |

## 金刮刮卡（5 个奖励槽，同类型加成）

| 奖励类型 | 权重 | 概率 | 奖金（元） |
|----------|------|------|------------|
| 空奖     | 43   | 43%  | 0          |
| 小奖     | 27   | 27%  | 0.30       |
| 中奖     | 16   | 16%  | 1.00       |
| 大奖     | 11    | 11%   | 3.00       |
| 极大奖   | 3    | 3%   | 16.00      |

## 钻石刮刮卡（5 个奖励槽 + 3 个倍率槽，同类型加成 + 倍率乘积）

奖励槽概率同金刮刮卡。

倍率槽（每个独立）：

| 倍率值 | 权重 | 概率 |
|--------|------|------|
| 1×     | 55   | 55%  |
| 2×     | 35   | 35%  |
| 3×     | 7    | 7%   |
| 4×     | 2.2  | 2.2% |
| 5×     | 0.8  | 0.8% |

### 概率计算方式

单个奖励类型的概率 = 该类型权重 ÷ 所有奖励类型权重之和 × 100%
